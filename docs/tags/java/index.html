Cấu trúc cơ bản: Try - Catch - Finally Try: Chứa đoạn code “nguy hiểm” có khả năng gây lỗi. Catch: Nơi xử lý khi lỗi xảy ra. Finally: Code trong khối này LUÔN LUÔN chạy, dù có lỗi hay không (Thường dùng để đóng kết nối DB, đóng file). 2. Phân biệt throw và throws throw: Hành động ném ra một lỗi cụ thể (Dùng bên trong thân hàm). Ví dụ: if (age < 18) throw new ArithmeticException("Chưa đủ tuổi"); throws: Khai báo rằng hàm này có thể gây ra lỗi, người gọi hàm phải tự xử lý (Dùng ở chữ ký hàm).
